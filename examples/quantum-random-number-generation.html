Rabdomness Extraction for Random Number Generation

In quantum random number generation (QRNG), randomness extraction is used to ensure the final output is only from the randomness certified by some estimation process, which accounts for adversarial information. Here, we give an example of how our library can be used off-the-shelf in this scenario.

Semi-Device-Independent Heterodyne-Based QRNG

We consider the semi-device-independent quantum random number generator based on heterodyne detection [ATVV2021]. This protocol is a semi-device-independent QRNG based on heterodyne detection that assumes an upper bound on the amount of energy generated states have, avoiding an identically and independently distribution type assumption on generated states. The protocol security is considered against classical side information only, so the randomness extractor need only be classical-proof.

The experimental demonstration uses the Toeplitz extractor, with an input length of \(n = 6500000000\), input min-entropy of \(k= 581295000\), output length of \(m = 581294933\), extractor error of \(\epsilon = 10^{-10}\) and seed length \(d = 7081294932\).

[1]:

import cryptomite
from math import log2, floor
def randomness_extraction(seed_bits, raw_randomness, n = 6500000000, m = 581294933):
    """ Perform randomness extraction for example QRNG.

    Parameters
    ----------
    seed_bits : list of (uniformly random) bits statistically
    independently of the bits generated in the protocol.
    raw_randomness : list of bits from
    the results of measurements in the QRNG
    protocol based on heterodyne measurements.
    n: integer, the number of raw randomness bits.
    m: integer, the number of extractable bits, given
    the error and min-entropy.

    Returns
    ---------
    list of bits
    The extracted output, which is
    the shared secret key.
    """
    toeplitz = cryptomite.Toeplitz(n, m)
    return toeplitz.extract(seed_bits, raw_randomness)

Extensions

The experimental demonstration above admits some improvements with cryptomite library:

Extension 1: Dodis Extraction

This change gives a significant reduction in the size of the required uniform seed and a marginally quicker implementation. There is a slight cost to this, which is adjusting \(n = 6.5 \cdot 10^9$\) to \(n^{\Dodis} = 6499999931\). This results in an output length of \(581'294'865$\) Overall, for a reduction of \(581'294'932\) uniform seed bits, the price to pay is a loss of \(68\) output bits. We implement this below.

[ ]:

import cryptomite
from math import floor, log2
def randomness_extraction_dodis(
        seed_bits,
        raw_key_bits,
        input_entropy = 581295000,
        error = log2(10**(-10))):
    n, m = cryptomite.Dodis.from_params(
        seed_length = len(seed_bits),
        input_length = len(raw_key_bits),
        seed_entropy = len(seed_bits),
        input_entropy = input_entropy,
        error = error)
    dodis = cryptomite.Dodis(n,m)
    return dodis.extract(seed_bits, raw_key_bits)
